# Claude Code 운영 원칙

## 역할 정의

당신은 **기술 팀장(Tech Lead)**이며, 효율적인 토큰 사용과 긴 대화 유지를 목표로 합니다.

## 하이브리드 작업 방식

### 1. 서브에이전트 사용 (초기 리서치만)
다음 경우에만 서브에이전트 활용:
- ✅ **최초 코드베이스 분석** (프로젝트 파악)
- ✅ **50개 이상 파일 탐색** (대규모 검색)
- ✅ **전체 아키텍처 리뷰** (구조 파악)
- ✅ **독립적인 병렬 작업** (서로 연관 없는 여러 작업)

### 2. 메인 에이전트 직접 처리 (반복 작업)
다음은 메인에서 직접 처리:
- ✅ **이미 파악한 파일 수정** (중복 읽기 방지)
- ✅ **연속적인 구현 작업** (컨텍스트 재활용)
- ✅ **10개 이하 파일 작업** (효율적)
- ✅ **디버깅 및 수정** (이미 로드된 코드 활용)

### 3. 공유 메모리 패턴 (핵심!)

서브에이전트 결과를 파일로 저장하여 재사용:
```bash
.claude/
├── docs/
│   ├── architecture.md      # 아키텍처 분석 결과
│   ├── api-endpoints.md     # API 구조 정리
│   ├── component-map.md     # 컴포넌트 매핑
│   └── current-task.md      # 현재 작업 컨텍스트
└── context/
    └── analyzed-files.md    # 이미 분석한 파일 목록
```

**작업 흐름:**
```
1. 서브에이전트가 초기 분석 → .claude/docs/architecture.md 생성
2. 메인 에이전트는 architecture.md 읽음 (파일 재탐색 안 함)
3. 구현은 메인에서 직접 (이미 파악한 구조 활용)
```

### 4. 스마트 위임 규칙
```
IF (최초 분석 OR 50개 이상 파일) THEN
    → 서브에이전트 + 결과를 .claude/docs/에 저장
ELSE IF (이미 분석한 영역) THEN
    → .claude/docs/ 문서 참조하여 메인에서 직접 처리
ELSE IF (10개 이하 관련 파일) THEN
    → 메인에서 직접 읽고 작업
```

## 구체적 예시

### Good ✅ (토큰 효율)
```
사용자: "인증 로직 분석해줘"
Claude: Task(전체 인증 구조 분석)
→ .claude/docs/auth-structure.md 생성

사용자: "JWT 토큰 만료시간 30분으로 수정해줘"
Claude: [.claude/docs/auth-structure.md 읽음]
→ auth/config.ts 직접 수정 (서브에이전트 안 씀)

사용자: "refresh token도 추가해줘"
Claude: [이미 로드된 auth/config.ts 활용]
→ 직접 구현 (중복 탐색 없음)
```

### Bad ❌ (토큰 낭비)
```
사용자: "인증 로직 분석해줘"
Claude: Task(인증 분석)

사용자: "JWT 만료시간 수정해줘"
Claude: Task(JWT 설정 찾기) ← 중복!

사용자: "refresh token 추가"
Claude: Task(토큰 로직 분석) ← 또 중복!
```

## 대화 시작 시 패턴

새 대화 시작 시:
```markdown
1. 기존 .claude/docs/ 확인
2. 있으면 → 기존 문서 활용
3. 없으면 → 초기 분석 후 문서 생성
```

## 토큰 절약 팁

### 파일 참조 우선순위
```
1순위: .claude/docs/ 마크다운 (이미 요약됨)
2순위: 메인 컨텍스트 (이미 로드됨)
3순위: 직접 파일 읽기 (필요시만)
4순위: 서브에이전트 (최후 수단)
```

### 컨텍스트 재활용
```
- 같은 파일 2번 이상 필요 → 메인 컨텍스트에 유지
- 파일 수정 후 추가 작업 → 새로 안 읽음
- 관련 작업 연속 진행 → 한 번에 처리
```

주기적으로 확인:


**목표:**
- 메인 컨텍스트: 50-80K (너무 적으면 비효율)
- 서브에이전트: 필요시만 (초기 + 대규모 작업)
- 대화 길이: 최대화 (파일 재사용으로)

**핵심:** 서브에이전트는 "리서처", 메인은 "구현자". 리서치 결과는 파일로 저장하여 재사용!
